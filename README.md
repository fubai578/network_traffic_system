# 网络流量分析与异常检测系统

## 一、系统概述

本系统为网络流量分析与异常检测系统，能够读取网络会话数据，构建通信关系图，并提供流量排序、路径查找、异常拓扑检测、安全规则检查等功能。系统提供命令行与图形界面两种使用方式，并支持将子图以交互式网页的形式可视化展示。

---

## 二、编译

**环境要求：** g++ 支持 C++17 及以上

在项目根目录下执行：

```bash
g++ -std=c++17 -O2 -o ntas src/*.cpp -Iinclude
```

编译成功后生成可执行文件 `ntas`（Windows 下为 `ntas.exe`）。

---

## 三、运行环境要求

**软件要求：**
- 操作系统：Windows 10 / 11（当前发布版本已在 Windows 下验证）
- Python 3.8 及以上版本（使用图形界面或可视化功能时需要）
- 所需 Python 第三方库：`tkinter`（Python内置）、`networkx`、`ipysigma`、`scapy`（仅pcap转换功能需要）

## 五、启动方式

### 方式一：图形界面（推荐）

双击运行 `py_gui.py`，或在命令提示符中执行：

```
python py_gui.py
```

启动后界面上方为功能按钮区，中部为输出显示区，下方为输入框。

### 方式二：命令行模式

直接运行 `ntas.exe`，支持可选参数指定数据文件路径：

```
ntas.exe
ntas.exe data/network_data.csv
```

启动后在终端中按照提示输入数字选项进行操作。

## 六、功能使用说明

### 6.1  加载数据（选项 1）

这是使用所有其他功能的前提步骤，**每次启动程序后必须首先执行此操作**。

**操作步骤：**
1. 点击【读取CSV并构建图】按钮（或输入 `1`）
2. 在输入框中输入CSV文件路径，直接回车则使用默认路径 `data/network_data.csv`
3. 等待输出区显示加载成功信息，例如：

```
[CSV] Loaded 1264 records, skipped 0 invalid lines.
[图构建] 节点数: 120，边数（合并后）: 891
Data loaded and graph built successfully!
```

**注意：** 若提示"Failed to load data"，请检查文件路径是否正确，以及CSV文件格式是否符合要求（需包含 Source、Destination、Protocol、SrcPort、DstPort、DataSize、Duration 七列）。

---

### 6.2  节点流量排序（选项 3）

统计所有节点的收发总流量并降序排列，可用于快速定位高流量节点。

**操作步骤：**
1. 点击【节点流量排序】按钮（或输入 `3`）
2. 输入希望显示的排名数量，输入 `0` 显示全部节点
3. 输出区显示排名表格，包含节点IP、总流量、发出流量、接收流量

**输出示例：**
```
======== Node traffic ranking (top 10) ========
Rank IP                TotalBytes     OutBytes       InBytes
--------------------------------------------------------------------
1    192.168.1.105     8523100        7102300        1420800
2    10.0.0.23         6341200        980100         5361100
...
```

---

### 6.3  HTTPS节点流量排序（选项 4）

筛选出参与过HTTPS通信（TCP协议且目的端口为443）的节点，按流量排序输出，便于聚焦加密流量分析。

**操作步骤：**
1. 点击【HTTPS节点排序】按钮（或输入 `4`）
2. 输入希望显示的排名数量，输入 `0` 显示全部

---

### 6.4  单向流量节点排序（选项 5）

筛选发出流量占总流量比例超过指定阈值的节点，此类节点可能正在进行端口扫描等异常行为。

**操作步骤：**
1. 点击【单向流量节点】按钮（或输入 `5`）
2. 输入出流量占比阈值（0到1之间的小数，直接回车使用默认值 `0.8`）
3. 输入希望显示的排名数量

**输出示例：**
```
==== Unidirectional (outgoing >80%) nodes (total 7, show top 7) ====
(These nodes may indicate port scanning or other suspicious behavior)
Rank IP                TotalBytes     OutBytes    OutRatio%
--------------------------------------------------------------
1    172.16.0.88       3201400        2977300     93.0%
...
```

---

### 6.5  路径查找与对比（选项 6）

给定源IP和目的IP，同时计算拥塞最小路径（Dijkstra）与跳数最小路径（BFS），并对比输出两条路径的差异。

**操作步骤：**
1. 点击【路径查找与对比】按钮（或输入 `6`）
2. 在输入框中输入源IP地址，回车确认
3. 继续输入目的IP地址，回车确认
4. 输出区显示两条路径的详细对比

**输出示例：**
```
========== Path comparison: 10.0.0.1 -> 192.168.1.200 ==========

  [Plan A - Min-congestion (Dijkstra)]
    Path: 10.0.0.1 -> 10.0.0.45 -> 192.168.1.200
    Hops: 2  Congestion: 1523.45 bytes/s

  [Plan B - Min-hop (BFS)]
    Path: 10.0.0.1 -> 192.168.1.200
    Hops: 1  Congestion: 98234.10 bytes/s

  Paths differ: Plan A minimizes congestion; Plan B minimizes hops.
```

**注意：** 若提示"Node not found"，说明该IP地址不存在于当前数据集中，请检查IP是否输入正确。

---

### 6.6  星型拓扑检测（选项 7）

自动检测图中所有星型拓扑结构，无需额外输入。符合条件的定义为：中心节点与20个或以上节点相连，且这些节点只与该中心节点建立连接。

**操作步骤：**
1. 点击【星型拓扑检测】按钮（或输入 `7`）
2. 等待检测完成，结果自动输出

**输出示例：**
```
======== Star topology detection ========
Center: 10.10.10.1  Leaves: 34
10.10.10.2, 10.10.10.3, 10.10.10.4, ...
```

若输出"No star topology found."，则当前数据中不存在符合条件的星型结构。

---

### 6.7  安全规则检查（选项 8）

基于用户自定义的IP地址范围规则，检测违规会话。支持黑名单（DENY）和白名单（ALLOW）两种模式。

**操作步骤：**
1. 点击【安全规则检查】按钮（或输入 `8`）
2. 依次在输入框中输入以下内容：
   - 受控节点IP（addr1）：要检查的源IP
   - IP范围起始地址（addr2）
   - IP范围结束地址（addr3）
   - 动作类型：输入 `1` 为DENY（黑名单，禁止与该范围通信），输入 `0` 为ALLOW（白名单，只允许与该范围通信）

**输出示例：**
```
======== Security rule check ========
Rule: 192.168.1.10 DENY sessions with [10.0.0.1 ~ 10.0.0.255]

SrcIP             DstIP             Proto     Bytes       Reason
----------------------------------------------------------------------
192.168.1.10      10.0.0.23         TCP       45230       Send to IP in denied range
10.0.0.45         192.168.1.10      UDP       12100       Receive from IP in denied range

Total 2 violation(s) found.
```

---

### 6.8  子图可视化（选项 2，仅图形界面可用）

给定一个IP地址，将该节点所在的连通子图渲染为交互式网页，在默认浏览器中打开。节点大小与流量成正比，查询节点以红色高亮显示。

**操作步骤：**
1. 点击【展示子图】按钮（或输入 `2`）
2. 输入希望查询的IP地址
3. 等待输出区显示"子图可视化已在浏览器打开"，浏览器将自动弹出交互式图页面

**交互操作说明：**
- 鼠标滚轮：缩放视图
- 鼠标拖拽：平移视图
- 点击节点：高亮显示该节点及其连接边
- 悬停节点：显示IP地址信息

生成的HTML文件保存在程序目录下的 `graph/` 文件夹中，文件名依次为 `subgraph_1.html`、`subgraph_2.html`……不会覆盖历史记录。

---

### 6.9  pcap文件转CSV（独立工具）

如需从自行抓取的pcap文件中提取数据，可使用 `pcap_to_csv.py` 脚本进行格式转换。

**使用方法（在命令提示符中执行）：**

```
python pcap_to_csv.py capture.pcap
python pcap_to_csv.py capture.pcap output.csv
```

第一种方式输出到默认文件 `network_data.csv`；第二种方式可自定义输出路径。转换完成后，将生成的CSV文件通过选项1加载即可。

## 七、常见问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 点击按钮无响应 | C程序未正常启动 | 点击【重启C程序】按钮，查看是否提示找不到ntas.exe |
| "Please load data first" | 未执行数据加载 | 先点击【读取CSV并构建图】完成数据加载 |
| "Node not found" | IP地址不在数据集中 | 检查IP拼写，或先通过流量排序确认存在的IP |
| 可视化页面无法打开 | 缺少Python依赖库 | 执行 `pip install networkx ipysigma` |
| pcap转换失败 | 未安装scapy | 执行 `pip install scapy` |
| C程序已退出提示 | 输入`0`或程序异常退出 | 点击【重启C程序】重新启动 |
